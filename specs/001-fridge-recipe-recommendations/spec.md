# Feature Specification: 냉장고 재료 기반 레시피 추천

**Feature Branch**: `001-fridge-recipe-recommendations`

**Created**: 2026-07-13

**Status**: Draft

**Input**: User description: "냉장고에 있는 재료를 체크하면 AI가 만들 수 있는 메뉴와 레시피를 추천해주는 웹앱"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 보유 재료로 메뉴 추천받기 (Priority: P1)

사용자는 냉장고에 있는 재료를 선택하거나 입력하고, 해당 재료로 만들 수 있는 메뉴와 레시피를 추천받는다.

**Why this priority**: 재료를 활용할 메뉴를 찾는 것이 서비스의 핵심 가치이며, 이 흐름만으로도 유효한 최소 기능을 제공한다.

**Independent Test**: 사용자가 재료를 1개 이상 등록하고 추천을 요청하면, 만들 수 있는 메뉴 목록과 각 메뉴의 레시피를 확인할 수 있는지 테스트한다.

**Acceptance Scenarios**:

1. **Given** 사용자가 재료 입력 화면에 있음, **When** 보유 재료를 하나 이상 선택하거나 입력하고 추천을 요청함, **Then** 시스템은 해당 재료를 활용할 수 있는 메뉴 목록을 우선순위와 함께 보여준다.
2. **Given** 추천 메뉴 목록이 표시됨, **When** 사용자가 메뉴를 선택함, **Then** 시스템은 필요한 재료, 조리 순서, 예상 조리 시간과 인분을 포함한 레시피를 보여준다.
3. **Given** 사용자가 보유 재료를 입력함, **When** 일부 재료가 없어도 만들 수 있는 메뉴를 요청함, **Then** 시스템은 보유 재료와 추가로 필요한 재료를 구분해 표시한다.

---

### User Story 2 - 재료 목록 관리하기 (Priority: P2)

사용자는 입력한 재료를 추가, 삭제, 수정하고 다음 추천에 다시 사용할 수 있다.

**Why this priority**: 재료를 쉽게 관리해야 추천 결과를 반복해서 사용할 수 있고 입력 부담을 줄일 수 있다.

**Independent Test**: 재료를 추가하고 삭제한 뒤 현재 재료 목록이 정확히 반영되는지 확인한다.

**Acceptance Scenarios**:

1. **Given** 재료 입력 화면에 있음, **When** 사용자가 재료명을 입력하거나 목록에서 선택함, **Then** 재료가 현재 보유 목록에 추가된다.
2. **Given** 보유 재료 목록에 재료가 있음, **When** 사용자가 해당 재료를 삭제함, **Then** 재료가 목록에서 제거되고 이후 추천에서 보유 재료로 사용되지 않는다.
3. **Given** 사용자가 이전에 저장한 재료 목록이 있음, **When** 사용자가 다시 방문함, **Then** 저장된 재료 목록을 확인하고 수정할 수 있다.

---

### User Story 3 - 취향과 상황에 맞게 추천 좁히기 (Priority: P3)

사용자는 조리 시간, 난이도, 식사 유형과 같은 조건을 선택해 자신의 상황에 맞는 메뉴를 추천받는다.

**Why this priority**: 기본 추천 이후 선택지를 줄여 실제 조리로 이어질 가능성을 높이지만, 기본 추천 없이도 서비스의 핵심 가치를 제공할 수 있다.

**Independent Test**: 조건을 하나 이상 선택해 추천을 요청하고 결과가 선택 조건을 반영하는지 확인한다.

**Acceptance Scenarios**:

1. **Given** 보유 재료가 입력되어 있음, **When** 사용자가 조리 시간 또는 난이도 조건을 선택함, **Then** 추천 결과는 선택한 조건을 만족하는 메뉴를 우선적으로 보여준다.
2. **Given** 선택한 조건을 만족하는 메뉴가 부족함, **When** 사용자가 추천을 요청함, **Then** 시스템은 조건을 완전히 만족하지 않는 결과가 있을 경우 어떤 조건이 부족한지 설명한다.

### Edge Cases

- 보유 재료가 입력되지 않은 상태에서 추천을 요청하면 재료를 먼저 입력하도록 안내한다.
- 입력한 재료로 만들 수 있는 메뉴가 없으면 추가하면 좋은 재료와 대체 가능한 일반 재료를 안내한다.
- 같은 재료를 여러 번 입력하면 하나의 재료로 합치고 중복 표시하지 않는다.
- 재료명이 모호하거나 인식되지 않으면 사용자가 확인하거나 수정할 수 있도록 한다.
- AI 추천 결과를 불러오지 못하면 원인을 알기 쉬운 메시지와 재시도 방법을 제공한다.
- 알레르기 또는 식이 제한 정보가 설정된 경우 해당 조건에 맞지 않는 메뉴를 추천하지 않는다.
- 레시피에 사용되는 양이 보유량보다 많은 경우 부족한 양을 명확히 표시한다.

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 시스템은 사용자가 재료명을 직접 입력하거나 제공된 재료 목록에서 선택할 수 있게 해야 한다.
- **FR-002**: 시스템은 현재 보유 재료 목록을 표시하고 사용자가 재료를 추가, 수정, 삭제할 수 있게 해야 한다.
- **FR-003**: 시스템은 중복 재료를 하나의 항목으로 통합해야 한다.
- **FR-004**: 시스템은 사용자가 입력한 보유 재료를 기준으로 만들 수 있는 메뉴를 추천해야 한다.
- **FR-005**: 시스템은 각 추천 메뉴에 보유 재료, 추가 필요 재료, 예상 조리 시간, 난이도와 인분을 표시해야 한다.
- **FR-006**: 시스템은 사용자가 추천 메뉴를 선택하면 재료의 필요량과 조리 순서를 포함한 레시피를 제공해야 한다.
- **FR-007**: 시스템은 보유 재료만으로 만들 수 있는 메뉴를 우선적으로 표시해야 한다.
- **FR-008**: 시스템은 보유 재료가 부족한 경우 부족한 재료와 양을 구분해 표시해야 한다.
- **FR-009**: 시스템은 사용자가 조리 시간, 난이도, 식사 유형 등의 추천 조건을 선택할 수 있게 해야 한다.
- **FR-010**: 시스템은 선택된 추천 조건을 결과 정렬과 메뉴 설명에 반영해야 한다.
- **FR-011**: 시스템은 추천 결과가 없거나 추천을 생성할 수 없는 경우 원인과 사용자가 취할 수 있는 다음 행동을 안내해야 한다.
- **FR-012**: 시스템은 사용자가 알레르기 또는 식이 제한 정보를 설정하고 변경할 수 있게 해야 한다.
- **FR-013**: 시스템은 설정된 알레르기 또는 식이 제한에 위배되는 메뉴를 추천 결과에서 제외해야 한다.
- **FR-014**: 시스템은 로그인이나 영구 저장 없이 현재 브라우저 세션 동안 재료 목록과 설정을 유지해야 한다.
- **FR-015**: 시스템은 AI가 생성한 레시피가 참고용임을 표시하고, 알레르기나 건강 관련 판단은 사용자가 최종 확인하도록 안내해야 한다.

### Key Entities *(include if feature involves data)*

- **보유 재료**: 사용자가 현재 가지고 있다고 표시한 재료. 이름, 보유량 또는 단위, 등록 시점 정보를 가진다.
- **추천 조건**: 조리 시간, 난이도, 식사 유형, 알레르기 및 식이 제한처럼 추천 결과에 적용하는 사용자 조건이다.
- **메뉴 추천**: 보유 재료와 추천 조건을 바탕으로 제안된 메뉴. 일치 재료, 추가 재료, 조리 시간, 난이도, 인분을 포함한다.
- **레시피**: 선택한 메뉴를 조리하기 위한 재료 필요량, 조리 순서, 주의사항으로 구성된 안내다.
- **사용자 설정**: 현재 브라우저 세션 동안 유지되는 재료 목록과 추천 조건이다.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 신규 사용자의 90%가 첫 방문에서 재료를 하나 이상 입력하고 3분 이내에 추천 메뉴를 확인한다.
- **SC-002**: 유효한 재료가 입력된 추천 요청의 95%에서 사용자는 최소 3개의 메뉴 또는 추천 결과가 없는 이유를 10초 이내에 확인한다.
- **SC-003**: 추천 메뉴를 선택한 사용자의 90%가 필요한 재료와 조리 순서를 한 화면 흐름 안에서 확인한다.
- **SC-004**: 사용자의 80% 이상이 재료 목록 추가, 삭제, 수정 중 원하는 작업을 첫 시도에 완료한다.
- **SC-005**: 알레르기 또는 식이 제한을 설정한 사용자의 추천 결과에서 해당 제한을 위반하는 메뉴가 0건 표시된다.
- **SC-006**: 사용성 평가에서 사용자의 80% 이상이 추천 결과가 자신의 보유 재료와 상황에 적합하다고 응답한다.

## Assumptions

- 주요 사용자는 회원가입 없이도 핵심 추천 기능을 사용할 수 있는 일반 소비자다.
- 재료명은 텍스트 입력과 미리 정의된 목록 선택을 함께 지원하며, 사진으로 재료를 인식하는 기능은 이번 범위에 포함하지 않는다.
- 메뉴 추천은 한국 가정식과 일상적으로 구할 수 있는 재료를 우선 대상으로 한다.
- 사용자가 입력하는 보유량은 선택 사항이며, 보유량이 없으면 재료 보유 여부만으로 추천한다.
- 추천 결과는 일반적인 조리 참고 정보이며, 의료적·영양학적 조언을 대신하지 않는다.
- 재료 목록과 사용자 설정은 현재 브라우저 세션에서만 유지하며, 새로고침하거나 다시 방문하면 초기화한다.
- 초기 버전에서는 다른 사용자와 메뉴를 공유하거나 리뷰를 작성하는 기능을 제공하지 않는다.
